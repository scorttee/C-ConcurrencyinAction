# 笔记 

## 第一章 

### 何为并发 
1.计算机系统中的并发
	计算机领域的并发指的是在单个系统里同时执行多个独立的操作，而非顺序的执行一些活动。  
	实际上，很多因素会使得任务分割不均和调度不规则。
2.并发的途径 
	多进程并发
		将应用程序分为多个独立的进程，他们在同一时刻运行，通过常规的通信渠道进行通信 
		设置复杂，速度慢，运行多个进程所需的固定开销。
		更容易编写安全的并发代码。
		可以使用远程连接的方式，在不同机器上运行独立的进程。
	多线程并发
		使用多线程相关的开销远远小于使用多个进程。 
		地址空间共享，缺少线程间的数据保护。
		多线程是主流语言更青睐的并发途径。
3.并发与并行 
	两者的主要区别是：关注点和意图方面，并行更加关注性能，当关注重点是任务分离或任务相应时，就会讨论程序的并发性。 
	两者存在的意义仅在不同的关注点。
### 为什么使用并发 
  关注点分离（SOC）和性能
1.为了分离关注点
	将相关的代码与无关的代码进行分离，是程序更容易理解和测试。从而减少出错的可能性。
2.为了性能
	单个任务分成几部分，且各自并行运行，从而降低总运行时间，这就是任务并行。
		常被称为易并行算法，具有良好的可拓展性。
	每个线程在不同数据部分上执行相同的操作，称之为数据并行。
		使用可并行的方式，来解决更大的问题。
		这种方法所带来的吞吐量的提升，可以让某些功能成为可能。
3.什么时候不使用并发
	收益比不上成本。
	性能增益小于预期。
	线程的资源有限。
	运行越多的线程，操作系统就需要越多的上下文切换。
### C++中的并发和多线程
	随着时间的推移，C++对并发和并行提供了更多的支持
	提供了更高级别的抽象与工具，使得编写多线程代码更简洁，并且不易出错。
	C++标准库中没有提供所需的性能或行为时，就需要使用平台相关的工具。这就本质而言，超出了C++标准库的本身范围。
### 总结
	了解并发与多线程的含义，何时选用或不用它
	C++中使用多线程并不复杂，复杂的是如何设计代码以实现其预期的行为。
